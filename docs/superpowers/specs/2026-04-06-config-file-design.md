# Config File Design

**Date:** 2026-04-06
**Status:** Approved

## Overview

Introduce a TOML config file for To Wit, replacing the `TOWIT_DB_PATH` environment variable and laying the groundwork for future configuration options (project filtering, topic suppression, topics-per-conversation count, command defaults, etc.).

## Goals

- Give users a persistent, human-readable config file that survives database resets
- Replace `TOWIT_DB_PATH` env var with a config file entry
- Keep `TOWIT_CONFIG_PATH` as a test-friendly escape hatch
- Make config opt-in: absence of the file is not an error

## Non-Goals

- `TOWIT_SETTINGS_PATH` is not added to the config (it is test-only and would be circular)
- Future config features (project filtering, topic suppression, etc.) are not implemented now — only `[database] path` is implemented in this iteration

## Config File

**Default location:** `~/.towit/config.toml`
**Override:** `TOWIT_CONFIG_PATH` environment variable

**Initial schema:**

```toml
# To Wit configuration
# Generated by: towit setup --config

[database]
# path = "~/.towit/catalog.db"  # Default; uncomment to override
```

All options are commented out in the generated file, serving as self-documentation. The file is written with `0600` permissions (matching the database).

## Architecture: `towit_config.py`

A new shared module (`libexec/towit/towit_config.py`) is the single source of truth for configuration. It follows the existing pattern of thin shared modules like `towit_db.py`.

**Responsibilities:**
- Locate the config file (via `TOWIT_CONFIG_PATH` env var or default path)
- Load and parse the TOML file
- Warn on errors, fall back to defaults
- Expose a `Config` class with typed property accessors
- Export a module-level `config` singleton

**Interface:**

```python
CONFIG_PATH = os.environ.get('TOWIT_CONFIG_PATH', os.path.expanduser('~/.towit/config.toml'))

class Config:
    def __init__(self, path=CONFIG_PATH): ...

    @property
    def db_path(self) -> str:
        """Resolved database path from config, or default."""
        ...

config = Config()  # module-level singleton
```

**Loading behavior:**

| Condition | Behavior |
|---|---|
| Config file does not exist | Silent; all defaults apply |
| Config file has bad TOML syntax | Warning to stderr; all defaults apply |
| A key has wrong type | Warning to stderr for that key; default for that key only |
| Unknown key present | Warning to stderr; key ignored |

## Consumer Changes

Modules that currently resolve `TOWIT_DB_PATH` directly are updated to import `config.db_path` instead:

- `towit_db.py` — remove module-level `DB_PATH` env var resolution
- `towit_setup.py` — remove local `TOWIT_DB_PATH` lookup
- `towit_teardown.py` — remove local `TOWIT_DB_PATH` lookup
- `towit_implode.py` — remove local `TOWIT_DB_PATH` lookup

## `towit setup` Changes

Two flags gain config generation behavior:

- **`--config`** (new): Generates `~/.towit/config.toml` with all current options as commented-out defaults. If the file already exists, prints a message and exits without overwriting.
- **`--full`** (existing): Config generation is added as the first step, before setup + install-hook + backfill.
- **No flags**: Unchanged — does not generate a config file.

## `TOWIT_DB_PATH` Deprecation

If `TOWIT_DB_PATH` is set, `towit_config.py` detects it, emits a deprecation warning to stderr, and uses it as the db path for that run so existing setups don't break immediately. **Precedence:** `TOWIT_DB_PATH` takes priority over `[database] path` in the config file when both are present.

```
Warning: TOWIT_DB_PATH is deprecated. Set [database] path in ~/.towit/config.toml instead.
```

A migration note is added to `CHANGELOG.md` under the release that ships this change:

```toml
# ~/.towit/config.toml
[database]
path = "/your/custom/path/catalog.db"
```

No hard removal date is set; the warning is sufficient until decided otherwise.

## Test Strategy

**Existing tests:** Tests that set `TOWIT_DB_PATH` are updated to also set `TOWIT_CONFIG_PATH` pointing to a temp config file (or an empty one) to keep config loading isolated.

**New unit tests** (`tests/helpers/towit_config_test.py`):
- No config file → defaults returned
- Valid config with `[database] path` → value read correctly
- Bad TOML syntax → warning emitted, defaults returned
- Unknown key → warning emitted, known keys still work
- Wrong type for a key → warning for that key, default used
- `TOWIT_DB_PATH` set → deprecation warning emitted, value used
- `TOWIT_CONFIG_PATH` set → that file loaded instead of default

**Integration tests** (additions to `tests/bin/towit.bats`):
- `towit setup --config` generates the file
- `towit setup --config` is idempotent (second run does not overwrite)
- `towit setup --full` generates the config file as first step

## Python Version

`tomllib` is stdlib as of Python 3.11. This change bumps the documented minimum Python requirement to 3.11. If `tomllib` is unavailable (Python < 3.11), a clear error is raised:

```python
try:
    import tomllib
except ImportError:
    raise SystemExit("towit requires Python 3.11+ for TOML config support.")
```
